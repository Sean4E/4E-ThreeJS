Part 1
The HTML Document
The wrapper the browser reads before running any JavaScript or Three.js code.
DOCTYPE
Document Type Declaration
HTMLRequired — Line 1

The very first line of every HTML file. It is not a tag — it is an instruction to the browser: "parse this file as modern HTML5." Without it, some browsers fall into Quirks Mode and render things incorrectly.

<!-- Always the very first line, no whitespace before it -->
<!DOCTYPE html>
⚠ Not an element — no closing tag, no attributes to change. Leave it exactly as shown.
STRUCTURE
Document Tags — html, head, body
<html lang="en">

The root element. Everything lives inside it. The lang attribute declares the language for screen readers and search engines.

<head>

Contains metadata — information about the page the browser uses but doesn't display. Title, fonts, styles, and importmaps go here.

<body>

Everything the user sees and interacts with. Your canvas, divs, buttons, and module scripts all live inside here.

Nesting

Tags must be properly nested — a child tag must close before its parent closes. <div><p>text</p></div> ✓ — never overlap tags.

<!DOCTYPE html>
<html lang="en">
  <head> <!-- metadata --></head>
  <body> <!-- visible content --></body>
</html>
HEAD
Meta Tags & Title
<meta charset="UTF-8">

Sets the character encoding. UTF-8 supports every language and emoji. Without it, accented characters may render as garbage.

<meta name="viewport">

Controls mobile scaling. width=device-width matches the physical screen. initial-scale=1.0 disables automatic zoom.

<title>

Text shown in the browser tab, used by bookmarks and search engines. One per page, always inside <head>.

Void Elements

Meta tags are void elements — no content, no closing tag. Same applies to <link>, <br>, <img>, and <input>.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Three.js Scene</title>
</head>
LOADING
<script>, <link> and importmap
<script src="...">

Loads an external JavaScript file or library. Used to bring in Three.js from a CDN. Place at the bottom of <body> so HTML loads first.

type="module"

Marks the script as an ES Module, enabling import / export syntax. Required for modern Three.js. Execution is automatically deferred.

defer / async

defer — load in background, run after HTML is parsed. async — run as soon as ready, order not guaranteed. Use defer for Three.js.

<link rel="stylesheet">

Loads an external CSS file or Google Font. Lives in <head>. The rel attribute names the relationship of the resource.

importmap

A <script type="importmap"> block that maps bare module names to CDN URLs — lets you write import ... from 'three' cleanly in the browser.

CDN

Content Delivery Network — a remote server hosting a library. No local install needed. cdn.jsdelivr.net is commonly used for Three.js.

<head>
  <link rel="stylesheet" href="https://fonts.googleapis.com/...">
  <script type="importmap">
    { "imports": {
        "three": "https://cdn.jsdelivr.net/npm/three@0.160/build/three.module.js",
        "three/addons/": "https://cdn.jsdelivr.net/npm/three@0.160/examples/jsm/" }}
  </script>
</head>
<body><script type="module">
  import * as THREE from 'three';
  import { OrbitControls } from 'three/addons/controls/OrbitControls.js';
</script></body>
💡 A script tag is like plugging in a power tool — it does nothing until your code uses it. The CDN URL is just the cable.
CANVAS
The <canvas> Element
HTML5Three.js render target

A rectangular pixel-drawing surface built into the browser. Blank by default — JavaScript draws into it. Three.js can create its own canvas automatically, or attach to one you place in the HTML.

width / height attributes

Sets the internal pixel resolution. Separate from CSS display size — a mismatch causes blurry output.

id attribute

A unique name so JS can grab it with document.getElementById('id'). Pass to WebGLRenderer as the canvas option.

renderer.domElement

The canvas Three.js created internally. A plain <canvas> you can style with CSS. Append with document.body.appendChild().

WebGL Context

The low-level GPU drawing API inside the canvas. Three.js wraps this entirely — you never call WebGL directly.

<canvas id="scene"></canvas>
<script type="module">
  // Option A — Three.js creates and appends its own canvas
  const renderer = new THREE.WebGLRenderer({ antialias: true });
  document.body.appendChild(renderer.domElement);

  // Option B — attach to your existing canvas element
  const canvas = document.getElementById('scene');
  const r2     = new THREE.WebGLRenderer({ canvas });
</script>
Part 2
Three.js Core Objects
The six objects every Three.js scene requires — and what each one actually does.
SCENE
THREE.Scene
THREE.SceneRoot Container

The root container for your entire 3D world. Objects, lights, cameras — everything must be added to the scene before it can be rendered.

scene.add( object )

Places any Object3D (mesh, light, group) into the scene graph so it is drawn on the next render call.

scene.remove( object )

Removes an object from the scene. It still exists in memory — just no longer rendered until added back.

scene.background

Set to a Color, Texture, or CubeTexture. Default is null (transparent black).

scene.fog

THREE.Fog(color, near, far) adds distance haze. FogExp2 gives exponential falloff.

const scene      = new THREE.Scene();
scene.background = new THREE.Color('#0a0a14');
scene.fog        = new THREE.Fog(0x0a0a14, 10, 50);
scene.add(mesh); scene.add(light);
🎭 The Scene is the theatre building — it holds the stage, rigs and props. The camera is the audience looking in.
CAMERA
THREE.PerspectiveCamera
THREE.PerspectiveCameraTHREE.OrthographicCamera

The eye looking into the scene. Nothing renders without one. PerspectiveCamera mimics human vision — distant objects appear smaller. OrthographicCamera has no perspective — parallel lines stay parallel (isometric / UI / blueprints).

fov — Field of View

Vertical angle in degrees the camera sees. 50–75° looks natural. Lower = telephoto. Higher = wide-angle distortion.

aspect

Width-to-height ratio of the viewport. Almost always innerWidth / innerHeight. Must be updated on resize.

near — Near Clip Plane

The closest visible distance. Anything nearer is clipped (invisible). Never set to 0 — use 0.1 as a safe default.

far — Far Clip Plane

The farthest visible distance. Keep tight — enormous values reduce depth buffer precision and can cause Z-fighting.

camera.position

A Vector3. Default is the origin (0,0,0) — which puts it inside any object placed there. Use .position.set(x,y,z).

camera.lookAt( target )

Points the camera at a Vector3. Call after positioning. OrbitControls replaces this automatically when active.

const camera = new THREE.PerspectiveCamera(
  60,                                     // fov
  window.innerWidth / window.innerHeight,  // aspect
  0.1,                                    // near
  100                                     // far
);
camera.position.set(0, 2, 5);
camera.lookAt(0, 0, 0);
📷 PerspectiveCamera = a real lens with natural depth. OrthographicCamera = an architect's blueprint — no vanishing point.
RENDERER
THREE.WebGLRenderer
THREE.WebGLRendererGPU Bridge

The engine that draws the scene. Takes the scene and camera, uses the GPU via WebGL, and outputs pixels to the canvas. Without it, nothing appears.

antialias: true

Smooths jagged staircase edges on geometry. Small GPU cost, visibly cleaner result. Set in the constructor options object.

renderer.setSize(w, h)

Sets the output resolution of the canvas. Call once on setup and again in the resize listener.

renderer.setPixelRatio()

Matches the device's pixel density. Use Math.min(devicePixelRatio, 2) to avoid over-rendering on Retina screens.

renderer.render(scene, cam)

The actual draw call. Called every frame — takes a snapshot from the camera's viewpoint and paints it to the canvas.

renderer.domElement

The canvas element Three.js created. Append to the page with document.body.appendChild(renderer.domElement).

renderer.shadowMap.enabled

Must be true to activate shadows. Lights and meshes also need castShadow / receiveShadow set individually.

const renderer = new THREE.WebGLRenderer({ antialias: true });
renderer.setSize(window.innerWidth, window.innerHeight);
renderer.setPixelRatio(Math.min(devicePixelRatio, 2));
renderer.shadowMap.enabled = true;
document.body.appendChild(renderer.domElement);
GEOMETRY
BufferGeometry — The Shape
THREE.BufferGeometryShape Data / Vertices

Defines the shape of a 3D object — positions of every vertex, how they connect into triangles, and UV coordinates for textures. Carries no colour or shading — that belongs to the material.

BoxGeometry(w, h, d)

A rectangular cuboid. Optional extra segment params allow finer shader deformation.

SphereGeometry(r, wSeg, hSeg)

A sphere. Higher segment counts = rounder but more expensive to render.

PlaneGeometry(w, h)

A flat rectangle in the XY plane. Rotate 90° on X (Math.PI / 2) to make a floor.

CylinderGeometry(rTop, rBot, h)

A cylinder or cone. One radius to 0 makes a cone. Equal radii = perfect cylinder.

TorusGeometry(r, tube, rSeg, tSeg)

A doughnut ring. r = ring radius, tube = cross-section thickness.

Vertices & Triangles

All geometry is made of triangles. Vertices are corner points. Three.js stores them in typed array BufferAttributes on the GPU.

const box    = new THREE.BoxGeometry(1, 1, 1);
const sphere = new THREE.SphereGeometry(0.5, 32, 16);
const plane  = new THREE.PlaneGeometry(10, 10);
const torus  = new THREE.TorusGeometry(1, 0.3, 16, 100);
// Geometry = shape data only. Pair with a material to make a Mesh.
🣎 Geometry is the wireframe armature — the shape without any surface quality. Paint comes from the material.
MATERIAL
Material — The Surface
THREE.MaterialColour / Shading / PBR

Defines how a surface looks — colour, shininess, transparency, and how it reacts to light. The same geometry looks completely different with different materials.

MeshBasicMaterial

No lighting. Always shows full colour regardless of lights in the scene. Wireframes, debug objects, always-visible glows.

MeshStandardMaterial

The physically based (PBR) default. Reacts to lights realistically. Has roughness and metalness. Use for most objects.

MeshPhysicalMaterial

Extended PBR with clearcoat, transmission (glass), and iridescence. More GPU-expensive than Standard.

MeshNormalMaterial

Colours each face by its normal direction. No lights needed — invaluable for debugging geometry orientation.

roughness / metalness

roughness 0 = mirror. roughness 1 = chalk. metalness 0 = plastic. metalness 1 = polished metal.

transparent / opacity

Set transparent: true then control opacity (0–1). Both required — opacity alone with no transparent flag will not work.

const mat = new THREE.MeshStandardMaterial({
  color: 0x8b5cf6, roughness: 0.3, metalness: 0.6
});
mat.color.set('#38bdf8'); // change colour after creation
mat.wireframe = true;
✓ MeshBasicMaterial — no lights needed. MeshStandardMaterial — needs at least one light or the object renders black.
MESH
THREE.Mesh — The Complete Object
THREE.MeshGeometry + Material combined

A Mesh is the complete renderable object — geometry (shape) plus material (surface) combined, placed into the scene. It inherits position, rotation, and scale from Object3D.

Object3D (base class)

Base for all positionable things. Cameras, lights, groups and meshes all share position, rotation, scale, name, and children.

mesh.name

String identifier. Find it later with scene.getObjectByName('cube'). Not required but very useful in complex scenes.

mesh.visible

Boolean. false hides the mesh without removing it — it still exists in memory and the scene graph.

castShadow / receiveShadow

Both must be true on relevant meshes, plus the renderer and light must have shadows enabled — all four required.

const cube = new THREE.Mesh(
  new THREE.BoxGeometry(1,1,1),
  new THREE.MeshStandardMaterial({ color: 0x8b5cf6 })
);
cube.name = 'mainCube';
cube.castShadow = true;
scene.add(cube);

// Clean up GPU memory when removing
scene.remove(cube);
cube.geometry.dispose();
cube.material.dispose();
🎲 Geometry = the cardboard shape. Material = the paint. Mesh = finished object — both combined, placed in the world.
LIGHTS
Lights — Illuminating the Scene

Required for physically-based materials to show colour and shading. MeshBasicMaterial ignores all lights. Every light must be added to the scene like any other object.

AmbientLight(color, intensity)

No direction, no shadows. Flat brightness on every surface. Used to lift shadows — think room bounce light.

DirectionalLight(color, intensity)

Parallel rays from one direction — like the sun. Position matters. Can cast sharp shadows.

PointLight(color, intensity, dist)

Radiates in all directions from one point — like a light bulb. Has a distance falloff and decay param.

SpotLight(color, intensity, dist, angle)

A cone of light with position and direction — like a stage spotlight. Most flexible shadow caster.

HemisphereLight(sky, ground, intensity)

Two colours — one from above, one from below. Cheap natural-looking ambient fill. Cannot cast shadows.

intensity

Brightness of the light. Default 1. Values above 1 are valid. Newer Three.js uses real-world luminance units.

const ambient = new THREE.AmbientLight(0xffffff, 0.4);
scene.add(ambient);

const sun = new THREE.DirectionalLight(0xfff5cc, 1.5);
sun.position.set(5, 10, 5);
sun.castShadow = true;
scene.add(sun);
Part 3
Transforms, The Loop & Extras
Positioning objects in space, animating them every frame, and utility tools that complete the toolkit.
TRANSFORMS
Position, Rotation & Scale

Every Object3D has three transform properties controlling where it lives in 3D space. These are the most-used properties in any Three.js project.

position — Vector3

The object's location in world space. position.set(x,y,z) or set individual axes. Three.js uses a right-handed, Y-up coordinate system.

rotation — Euler

Rotation in radians per axis. Math.PI = 180°, Math.PI / 2 = 90°. Rotation order matters — default is 'XYZ'.

scale — Vector3

Scale multiplier per axis. Default (1,1,1). scale.setScalar(n) for uniform scaling.

Vector3

Represents a point or direction in 3D space (x, y, z). Used for positions, directions, distances — the most common Three.js utility type.

Euler

Rotation as three angles in radians plus a rotation order string. Simpler to work with than Quaternions for everyday use.

Quaternion

4-component rotation that avoids gimbal lock. Used internally by Three.js. Access via mesh.quaternion. Prefer Euler for manual entry.

// Position
cube.position.set(2, 0.5, -1);
cube.position.x = 3;

// Rotation — always in radians
cube.rotation.y    = Math.PI / 4;  // 45°
cube.rotation.set(0, Math.PI, 0); // 180° facing away

// Scale
cube.scale.set(1, 2, 1);        // tall and thin
cube.scale.setScalar(1.5);      // uniform 1.5x

// Axes:  +X right   +Y up   +Z toward the viewer
📐 Position = GPS coordinates. Rotation = compass bearing + tilt. Scale = size multiplier before render.
LOOP
The Render Loop — requestAnimationFrame

requestAnimationFrame is a browser API that calls a function just before each screen repaint — typically 60fps. It is the heartbeat of any Three.js animation: update the scene, render it, request the next frame.

requestAnimationFrame(fn)

Schedules fn to run before the next browser repaint. Automatically pauses when the tab is hidden — saves CPU and GPU.

renderer.render(scene, cam)

The actual draw call. Takes a snapshot from the camera's viewpoint and paints it to the canvas. Called every frame.

THREE.Clock

clock.getElapsedTime() = total seconds. clock.getDelta() = seconds since last call. Use for frame-rate-independent motion.

Delta Time

Time between frames. Multiply speeds by delta so animation runs at the same pace on all devices regardless of frame rate.

const clock = new THREE.Clock();

function animate() {
  requestAnimationFrame(animate);          // schedule next frame
  cube.rotation.y = clock.getElapsedTime() * 0.5;
  controls.update();
  renderer.render(scene, camera);
}
animate(); // call once — recursion handles the rest
📌 Never call animate() more than once. The requestAnimationFrame inside it handles all subsequent frames automatically.
CONTROLS
OrbitControls & Window Resize
OrbitControls

An add-on (not in core Three.js) that lets users orbit, zoom, and pan with mouse or touch. Imported separately from three/addons/.

controls.enableDamping

Adds inertia — camera eases to a stop. Requires controls.update() every frame in the render loop.

controls.target

The orbit centre point. Default is the origin. Assign a Vector3 to orbit a different point.

Window Resize Handler

On resize, camera aspect and renderer output go stale. Always listen for window.resize and update both.

import { OrbitControls } from 'three/addons/controls/OrbitControls.js';

const controls = new OrbitControls(camera, renderer.domElement);
controls.enableDamping = true;
controls.dampingFactor  = 0.05;

window.addEventListener('resize', () => {
  camera.aspect = window.innerWidth / window.innerHeight;
  camera.updateProjectionMatrix();  // MUST call after changing aspect
  renderer.setSize(window.innerWidth, window.innerHeight);
});
⚠ Forgetting camera.updateProjectionMatrix() after changing camera.aspect silently keeps the old projection — one of the most common beginner bugs.
EXTRAS
Groups, Textures, Raycaster & Dispose
THREE.Group

An Object3D with no geometry or material — a parent container. Move or rotate the group and all children follow.

TextureLoader

Loads an image (JPG, PNG, WebP) and converts it to a Texture assignable to material slots like material.map.

material.map

The diffuse colour texture slot. Assign a loaded texture to paint an image onto the mesh surface.

Raycaster

Shoots a ray from a 2D screen point into the 3D scene. Returns which objects it hits — used for mouse click detection on 3D objects.

Scene Graph / Hierarchy

The parent–child tree of objects. A child's transforms are relative to its parent. Scene is the root node.

geometry.dispose()

Frees GPU memory. JS heap is automatic but Three.js GPU buffers must be freed manually with .dispose().

// Group — move multiple objects as one
const group = new THREE.Group();
group.add(cube, sphere);
scene.add(group);
group.rotation.y += 0.01; // rotates all children

// TextureLoader
const tex = new THREE.TextureLoader().load('./texture.jpg');
material.map = tex;
material.needsUpdate = true;

// Raycaster — pick objects with mouse
const ray   = new THREE.Raycaster();
const mouse = new THREE.Vector2();
window.addEventListener('click', e => {
  mouse.x =  (e.clientX / innerWidth)  * 2 - 1;
  mouse.y = -(e.clientY / innerHeight) * 2 + 1;
  ray.setFromCamera(mouse, camera);
  const hits = ray.intersectObjects(scene.children);
  if (hits.length) console.log('hit:', hits[0].object.name);
});
✓ For every new THREE.XXXGeometry() and new THREE.XXXMaterial(), there should be a matching .dispose() when the object is removed.
TEMPLATE
Minimal Working Scene

The absolute minimum to get a spinning, lit cube running in a browser. Every Three.js project expands outward from this.

starter-template.html 
<!DOCTYPE html><html lang="en"><head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width,initial-scale=1">
  <title>Three.js Starter</title>
  <style>* { margin: 0 } body { overflow: hidden }</style>
  <script type="importmap">
    { "imports": {
        "three": "https://cdn.jsdelivr.net/npm/three@0.160/build/three.module.js",
        "three/addons/": "https://cdn.jsdelivr.net/npm/three@0.160/examples/jsm/" }}
  </script></head>
<body><script type="module">
  import * as THREE from 'three';
  import { OrbitControls } from 'three/addons/controls/OrbitControls.js';

  // 1 · Scene
  const scene    = new THREE.Scene();
  // 2 · Camera
  const camera   = new THREE.PerspectiveCamera(60, innerWidth/innerHeight, 0.1, 100);
  camera.position.set(0, 1, 4);
  // 3 · Renderer
  const renderer = new THREE.WebGLRenderer({ antialias: true });
  renderer.setSize(innerWidth, innerHeight);
  renderer.setPixelRatio(Math.min(devicePixelRatio, 2));
  document.body.appendChild(renderer.domElement);
  // 4 · Mesh
  const cube = new THREE.Mesh(
    new THREE.BoxGeometry(),
    new THREE.MeshStandardMaterial({ color: 0x8b5cf6, roughness: 0.3 })
  );
  scene.add(cube);
  // 5 · Lights
  scene.add(new THREE.AmbientLight(0xffffff, 0.5));
  const sun = new THREE.DirectionalLight(0xffffff, 1.5);
  sun.position.set(5, 5, 5); scene.add(sun);
  // 6 · Controls
  const controls = new OrbitControls(camera, renderer.domElement);
  controls.enableDamping = true;
  // 7 · Resize
  window.addEventListener('resize', () => {
    camera.aspect = innerWidth / innerHeight;
    camera.updateProjectionMatrix();
    renderer.setSize(innerWidth, innerHeight);
  });
  // 8 · Loop
  const clock = new THREE.Clock();
  (function animate() {
    requestAnimationFrame(animate);
    cube.rotation.y = clock.getElapsedTime() * 0.5;
    controls.update();
    renderer.render(scene, camera);
  })();
</script></body></html>

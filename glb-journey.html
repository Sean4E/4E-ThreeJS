Blender to Browser
Six Steps to GLB Import

Each step adds one idea. By the end, your Blender models live in the browser.

01

Your Starting Point

You already have a working 3D scene from Week 7 — a shape, materials, lights, orbit controls, and animation. This journey adds one thing: loading real Blender models.

Your Week 7 scene is the stage. Now we're bringing in the actors — your own 3D models from Blender.
Step 1 — The imports (note the new one) 
import * as THREE from 'three';

import { OrbitControls } from
  'three/addons/controls/OrbitControls.js';

The first two imports are from Week 7. The third — GLTFLoader — is the tool that reads .glb files and turns them into Three.js objects.

02

Add a File Picker

To load GLB files locally (without a server), we let the user pick the file with an HTML input. This bypasses CORS — the same trick we used for textures in Week 7.

Like an "Open File" dialog in any app. The browser reads the file directly — no upload, no server.
Step 2 — HTML + event listener 
<!-- In your HTML, before the canvas -->
<input type="file" id="pick"
       accept=".glb,.gltf">

// In your script
document.getElementById('pick')
  .addEventListener('change', function(e) {
    const file = e.target.files[0];
    if (!file) return;
    console.log('Selected:', file.name);
    // Next step: read the file...
  });

accept=".glb,.gltf" filters the file dialog to only show 3D model files. The change event fires when the user picks a file.

03

The Loader

GLTFLoader is Three.js's tool for reading GLB/glTF files. Import it just like OrbitControls — from three/addons/. The .parse() method takes raw binary data instead of a URL, which is how we bypass CORS.

GLTFLoader understands Blender's Principled BSDF materials. If it looks right in Blender's Material Preview, it will look right in Three.js.
Step 3 — FileReader + GLTFLoader.parse 
import { GLTFLoader } from
  'three/addons/loaders/GLTFLoader.js';

// Inside your file change listener:
const reader = new FileReader();
reader.onload = function() {
    const loader = new GLTFLoader();
    loader.parse(reader.result, '',
      function(gltf) {
        scene.add(gltf.scene);
      }
    );
};
reader.readAsArrayBuffer(file);
FileReader reads the file from disk
→ gives us an ArrayBuffer (raw binary data)
GLTFLoader.parse() converts binary
→ into Three.js objects (meshes, materials, animations)
gltf.scene is a Group containing all meshes
→ scene.add(gltf.scene) puts them in your world

No live demo here — we combine everything in the next step.

05

Make It Fit

Your Blender model might be too big, too small, or off-center when it first loads. Adjust it with the same transforms you used in Week 7 — position, rotation, scale.

Same as Blender's G, R, S keys — but with numbers.
Step 5 — Transform after loading 
loader.parse(reader.result, '',
  function(gltf) {
    const model = gltf.scene;

    // Scale (try different values!)
    model.scale.set(
      1.0,
      1.0,
      1.0
    );

    // Position
    model.position.set(0, 0, 0);

    // Rotate (in radians)
    model.rotation.y = Math.PI / 4;

    // Auto-centre
    const box = new THREE.Box3()
      .setFromObject(model);
    const center = box.getCenter(
      new THREE.Vector3());
    model.position.sub(center);

    scene.add(model);
  }
);

06

Animations

If your GLB has animations (from Blender keyframes), you can play them with AnimationMixer. This is optional — only works if your model actually has animations baked in.

The animations you created in Week 4 (walk cycles, rotations, bounces) export with the GLB. AnimationMixer plays them back frame by frame.
Step 6 — AnimationMixer 
let mixer; // declare outside

loader.parse(reader.result, '',
  function(gltf) {
    scene.add(gltf.scene);

    // Play animations if they exist
    if (gltf.animations.length > 0) {
      mixer = new THREE.AnimationMixer(
        gltf.scene
      );
      const clip = gltf.animations[0];
      mixer.clipAction(clip).play();
    }
  }
);

// In your animation loop:
const clock = new THREE.Clock();
function animate() {
    requestAnimationFrame(animate);
    if (mixer) mixer.update(
      clock.getDelta()
    );
    controls.update();
    renderer.render(scene, camera);
}

Bonus
Animation Controls

Two extras. Control playback and manage multiple animations.

B1

Play / Pause

AnimationMixer actions have a paused property. Toggle it to pause and resume. You can also adjust timeScale to speed up or slow down the animation.

Like the play/pause button on a video. The animation freezes in place and picks up exactly where it left off.
Bonus 1 — Play/Pause + Speed 
// Store the action so we can control it
let currentAction;

// When loading animations:
if (gltf.animations.length > 0) {
  mixer = new THREE.AnimationMixer(model);
  const clip = gltf.animations[0];
  currentAction = mixer.clipAction(clip);
  currentAction.play();
}

// Toggle play/pause
function togglePause() {
  if (!currentAction) return;
  currentAction.paused = !currentAction.paused;
}

// Change speed (0.5 = half, 2 = double)
function setSpeed(speed) {
  if (!currentAction) return;
  currentAction.timeScale = speed;
}

// Stop completely (resets to start)
function stopAnimation() {
  if (!currentAction) return;
  currentAction.stop();
}

B2

Multi-Animation

Many GLB files have multiple animations — walk, run, idle, jump. You can play them individually, combine them, or play all at once. Each animation becomes a separate clipAction on the mixer.

Like a playlist. Each animation is a track — you can solo one, mute others, or play them all together.
Bonus 2 — Multiple Animations 
// Store all actions
const actions = [];

// Create an action for EACH animation
gltf.animations.forEach(function(clip) {
  const action = mixer.clipAction(clip);
  actions.push({
    name: clip.name,
    action: action,
    playing: false
  });
});

// Toggle a single animation on/off
function toggleAnim(index) {
  const a = actions[index];
  if (a.playing) {
    a.action.stop();
    a.playing = false;
  } else {
    a.action.play();
    a.playing = true;
  }
}

// Play all at once
function playAll() {
  actions.forEach(function(a) {
    a.action.play();
    a.playing = true;
  });
}

// Stop all
function stopAll() {
  actions.forEach(function(a) {
    a.action.stop();
    a.playing = false;
  });
}

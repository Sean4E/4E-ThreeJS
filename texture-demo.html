Load an Image

01

Load an Image Texture

In Blender you drag an image into the Shader Editor. In Three.js you use TextureLoader — same idea, different tool.

TextureLoader reads a PNG/JPG file → creates a Texture object → you pass it to a material's map property → Three.js wraps the image around the geometry using UV coordinates.

This is the same as connecting an Image Texture node to the Base Color input of Principled BSDF. The map property IS the Base Color texture slot.
Load & apply a texture 
// 1. Create a texture loader
const loader = new THREE.TextureLoader();

// 2. Load the image file
const texture = loader.load("Week2A_grid.png");

// 3. Use it as the material's map
const material = new THREE.MeshStandardMaterial({
  map: texture
});

// 4. Create a mesh — texture wraps
//    around the shape automatically
const mesh = new THREE.Mesh(
  new THREE.PlaneGeometry(2, 2),
  material
);
scene.add(mesh);

02

UV Mapping in Action

Different shapes wrap the same texture in different ways. A Plane shows it flat. A Box repeats it on each face. A Sphere stretches at the poles. This is UV mapping — the same concept as in Blender.

In Blender you see UVs in the UV Editor. Three.js geometries have UVs built in — they're generated automatically when you create the shape.
Same texture, different shape 
// The texture stays the same
const texture = loader.load("Week2A_grid.png");
const mat = new THREE.MeshStandardMaterial({
  map: texture
});

// Only the geometry changes
const mesh = new THREE.Mesh(
  // Try each of these:
  new THREE.PlaneGeometry(1.6, 1.6),
  // new THREE.BoxGeometry(1.2, 1.2, 1.2),
  // new THREE.SphereGeometry(0.8, 48, 48),
  // new THREE.CylinderGeometry(0.6,0.6,1.2,48),
  // new THREE.TorusGeometry(0.6, 0.25, 32, 64),
  // new THREE.ConeGeometry(0.7, 1.2, 48),
  mat
);
scene.add(mesh);

Create with Code

03

Canvas Texture

Instead of loading a file, you draw a texture with JavaScript. Create a hidden HTML canvas, draw on it with the Canvas 2D API, then wrap it in a CanvasTexture.

The material doesn't care where the texture came from — a PNG file or JavaScript code. Both produce a Texture that goes into the map slot.

Why bother? Canvas textures are dynamic — change colours on the fly, respond to user input, generate patterns from data. No image file to manage or load.
Draw a texture with code 
// 1. Create a hidden canvas
const cv = document.createElement("canvas");
cv.width = 512;
cv.height = 512;
const ctx = cv.getContext("2d");

// 2. Draw rainbow stripes
const colors = [
  "#ff0000", "#ff9900", "#ffff00",
  "#00ff00", "#0000ff",
  "#4b0082", "#8b00ff"
];
const h = cv.height / colors.length;
colors.forEach((col, i) => {
  ctx.fillStyle = col;
  ctx.fillRect(0, i * h, cv.width, h);
});

// 3. Turn it into a Three.js texture
const texture = new THREE.CanvasTexture(cv);

// 4. Use it exactly like an image texture
const material = new THREE.MeshStandardMaterial({
  map: texture
});
